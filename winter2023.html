January 17

Writing out the algorithm for enumerating trees.

Required:
All ISA trees T possible from a given mutation set M.
A number k representing how many losses you want inserted
A mutation m- to lose.

Output: All possible trees with mutation set M and k losses of m-

Algorithm 1:

  1. Create a list B
  2. For each tree t in the set of T
  3. Get all valid trees S where k m- mutations are inserted into t (Algorithm 2)
  4. Add all trees from S into B
  5. Return B
Required:
A number k of mutations to lose.
A Newick string representing a tree t.
A mutation m- that is being lost.
A list L of valid trees.

Output: N/A
Algorithm 2:
  1. If k is 0, add t to L if it's not already.
  2. Otherwise...
  3. Get the nodes N of t
  4. For node n in N
  5. If the node n is a valid node...
  6. Insert a loss of mutation m- to the node n such that m- has no children>
  7. Call Algorithm 2 with parameters...
      k-1 mutations to lose
      A new Newick string P* representing m- inserted onto node n
      A mutation m- that is being lost
      A list L of valid trees
  8. For each child c of n
  9. Insert a loss of mutation m- to the node n such that c is the child of m->
  10. Call Algorithm 2 with parameters...
      k-1 mutations to lose
      A new Newick string P* representing m- inserted onto node n
      A mutation m- that is being lost
      A list L of valid trees

February 20

Creating a simulation algorithm for multiple gains.

What are the constraints placed on a tree with multiple gains (no losses)?

  • You can't have two gains of same mutation on the same lineage
  • You can't have the same two gains as siblings

In reviewing my algorithm for enumerating trees, there's a likely chance that I'm overcounting as I didn't account for the case when two different base trees can end up having the same tree once you add gains/losses. Not sure if this was a problem for losses, but it does seem to be a problem for gains.
